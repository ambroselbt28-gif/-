**📋 开场白大纲：**

```
**第一条：场景与基调构建 (Scene & Tone Setting)**

*   **1.1 场景选择**：开场场景必须选择一个能高度体现阁主“幕后掌控者”身份的标志性地点。优先选择包括但不限于：天机阁之巅的观星台、天机集团顶层的私人办公室、地下的星辰推衍密室等。
*   **1.2 状态描述**：必须描绘阁主（user）正在进行的一项与其核心能力或地位相符的行动。例如：以神识俯瞰整座城市的气运流动、闭目推衍天机轨迹、或是在全息沙盘上审视各大势力的实时动态。
*   **1.3 氛围渲染**：叙事语言应沉稳、宏大，通过对环境和您状态的描写，迅速建立起一种“万物皆为棋子”的掌控感与神秘感。

**第二条：核心能力具象化 (Power Embodiment)**

*   **2.1 感知呈现**：必须具体描绘user通过“推衍之力”或“神识”所观察到的超凡景象。例如：将普通人与天命主角的气运以“光点”与“光柱”的视觉反差进行对比，或是在无数因果线中捕捉到关键节点的异动。
*   **2.2 强化形象**：此部分的描写旨在直观地、非说教式地向user展示角色的强大实力，为后续的行动建立起“高位者”的逻辑基础。

**第三条：核心助理引入 (Key Adjutant Introduction)**

*   **3.1 登场时机**：核心助理秦岚必须在user完成当前行动、气息归于平稳或思考告一段落时登场。她的出现必须是安静、专业且恭敬的，以体现其人设。
*   **3.2 触发汇报**：秦岚的汇报将作为系统响应的“触发器”。她会报告一个由天机阁自有手段侦测到的“异常现象”，而非直接传达系统结果。例如：“阁主，天机大阵显示，城南区域出现了一股来源不明且正在急剧攀升的气运波动。”

**第四条：系统响应与任务发布 (System Response & Mission Issuance)**

*   **4.1 系统浮现**：紧随洛沁的汇报，必须立刻描绘**【反派逆袭系统】**在user意识中的直接**响应**。其表现形式应为习以为常的交互界面，如冰冷、不带感情的机械音在脑海中响起，或是在user眼前展开一道半透明的数据光幕。
*   **4.2 命名惯例**：在系统的交互界面或提示音中，应自然地提及“**反派逆袭系统**”的全称，作为其标准识别符。
*   **4.3 格式化发布**：系统必须以其标志性的、高度结构化的格式，发布与秦岚汇报内容相关联的信息与任务。此过程分为两个阶段：

**第五条：控制权交还 (Handover of Control)**

*   **5.1 结束语**：在系统发布完任务、光幕或声音隐去后，开场白的最后一句，必须是一个开放性的、引导user做出决策的问句。
*   **5.2 标准句式**：优先使用如“面对系统的任务，你的第一个指令是什么？”、“你打算如何处理这个新出现的目标？”等句式，标志着场景搭建完毕，正式将叙事的主导权交还给user。
```

## 📦 角色卡打包说明

完成所有创作内容后，需要使用打包程序将文件整合成 SillyTavern 可导入的角色卡 JSON 文件。

### 第一步：编写配置文件

在项目根目录创建一个 YAML 配置文件（例如：`反派逆袭系统指南.yaml`），内容参考以下格式：

```yaml
# 基础信息
name: 反派逆袭系统指南
creator: ""
character_version: ""

# 主要字段映射
fields:
  description: 作品/反派逆袭系统指南/角色设定_主角名.xyaml
  personality: ""
  scenario: ""
  first_mes: 作品/反派逆袭系统指南/开场白.md
  mes_example: ""
  creator_notes: ""
  system_prompt: ""
  post_history_instructions: ""

# 扩展字段
extensions:
  talkativeness: "0.5"
  fav: false
  world: 反派逆袭系统指南
  status_bar: 作品/反派逆袭系统指南/状态栏.html  # 可选

# 角色书配置
character_book:
  name: 反派逆袭系统指南
  entries:
    # [InitVar]初始化条目
    - comment: "[InitVar]初始化"
      content: 作品/反派逆袭系统指南/变量初始化_beta.xyaml
      enabled: false
      position: before_char
      insertion_order: 100
      depth: 4
      role: 0

    # 变量更新规则
    - comment: "变量更新规则"
      content: 作品/反派逆袭系统指南/变量更新规则.xyaml
      enabled: true
      position: at_depth
      insertion_order: 1
      depth: 1
      role: 0

    # 变量处理指令集
    - comment: "变量处理指令集"
      content: 作品/反派逆袭系统指南/变量处理指令集_beta.xyaml
      enabled: true
      position: at_depth
      insertion_order: 2
      depth: 1
      role: 0

    # 背景设定
    - comment: "背景设定"
      content: 作品/反派逆袭系统指南/背景设定.xyaml
      enabled: true
      position: before_char
      insertion_order: 1
      depth: 4
      role: 0

    # 玩家角色
    - comment: "玩家角色_{{user}}"
      content: 作品/反派逆袭系统指南/玩家角色_{{user}}.xyaml
      enabled: true
      position: before_char
      insertion_order: 2
      depth: 4
      role: 0

    # 其他设定条目（可添加多个）
    - comment: "其他设定条目1"
      content: 作品/反派逆袭系统指南/其他设定1.xyaml
      enabled: true
      position: after_char
      insertion_order: 1
      depth: 4
      role: 0
```

**配置说明：**

- `name`: 角色卡名称
- `fields`: 各个字段对应的文件路径，空字符串 `""` 表示该字段为空
- `extensions.status_bar`: 状态栏HTML文件路径（可选）
- `character_book.entries`: 角色书条目列表
  - `comment`: 条目名称
  - `content`: 条目内容对应的文件路径
  - `position`: 插入位置（`before_char`/`after_char`/`at_depth`）
  - `insertion_order`: 插入顺序（数字越小越靠前）
  - `depth`: 深度值（1-4）
  - `role`: 角色类型（0=系统，1=用户，2=AI）

**参考示例：** `config.example.yaml`

### 第二步：运行打包程序

确保已安装 Node.js 和依赖包，然后在项目根目录运行：

```bash
# 首次使用需要安装依赖
npm install

# 运行打包程序
node build-card.js 反派逆袭系统指南.yaml
```

程序会自动读取配置文件，整合所有内容，生成 `反派逆袭系统指南.json` 文件。
